# Implementation Plan: Native Home Widgets

## Overview

Replace the fragile custom iOS Xcode target injection with `react-native-widget-extension`, create the iOS SwiftUI widget in `targets/widget/`, add Android Kotlin file copying to the config plugin, and move Kotlin source files to `plugins/android-widget/`. The existing `widgetBridge.ts` remains unchanged.

## Notes

- Tasks marked with `*` are optional and can be skipped for faster MVP
- `widgetBridge.ts` requires no code changes â€” the existing implementation already handles both platforms
- The three Android Kotlin files in `plugins/android-widget/` should match the current files in `android/app/src/main/java/com/anonymous/wedo/` but are sourced from the plugin directory so they survive `expo prebuild --clean`
- Property tests use `fast-check` which is already in devDependencies
- Each property test maps to a correctness property defined in the design document
